# Angel MMO 后续迭代路线

## 目标

Angel 当前是 MMO 服务端骨架，已经有多进程分层、WBUS 通信、对象 ID、DBCache 分层和共享内存/Lua 脚本入口。要成为可用 MMO 服务器，需要按“接入层 → 角色数据 → 场景状态 → 玩法系统 → 运维治理”的顺序演进。

## 迭代路线

### 1. 网关与账号

- Gateway 连接管理、心跳、限流、断线重连。
- 登录鉴权、账号态、封禁和踢人。
- Gateway 到 GameServer 的会话路由。

### 2. 角色与数据闭环

- 角色创建、选角、加载、保存。
- GameServer → DBManager → DBCache → DB 的读写链路。
- 脏数据标记、批量回写、异常恢复。

### 3. 场景与 AOI

- SceneServer 地图实例管理。
- 角色/NPC/掉落等对象进入、离开、迁移。
- AOI 可先用九宫格，后续按地图密度优化。
- 移动片段校验和广播。

### 4. 核心玩法

- 战斗、技能、Buff、背包、任务、邮件。
- 公会、好友、组队、匹配。
- 世界事件、排行榜、跨服玩法。

### 5. 共享内存对象化

- 角色、场景对象、事务对象统一接入共享内存对象生命周期。
- 稳定对象类型 ID、版本校验、恢复回调。
- 热重启时校验内存布局，失败时安全降级。

### 6. Lua 热更与配置

- Lua 配置读取和 Protobuf 映射。
- GM 函数执行、单文件热更、全量热更。
- 脚本执行权限、审计、回滚。

### 7. 运维与观测

- 一键起停服、灰度、滚动重启。
- 指标、日志聚合、链路追踪。
- 共享内存检查工具、DBCache 检查工具。
- 压测脚本和机器人客户端。

## 建议优先级

1. Gateway 登录链路 + GameServer 角色加载。
2. DBCache 持久化闭环。
3. SceneServer AOI 与对象同步。
4. 共享内存对象恢复与版本治理。
5. Lua 热更与 GM。
6. 公会、跨服、排行榜等中后期系统。
